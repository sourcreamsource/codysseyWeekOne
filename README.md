# 🟩 1주차: AI/SW 개발 워크스테이션 구축  

- 기술 문서(README.md) 작성 원칙  
    - 기술 문서(README.md)만 읽어도 전체 수행 내용을 파악할 수 있어야 한다.  
    - 모든 작업은 터미널(CLI) 기반으로 수행한다.  
    - 실행 결과(로그/접속/데이터 유지)로 핵심 흐름 넣기.  
    - 모든 기록을 알아보기 쉽게 다 기록한다.  
    - 이미지와 컨테이너의 분리, 격리된 실행 환경, 포트·스토리지 연결 방식이라는 구조적 원칙을 적용해 "왜 이런 설계가 필요한지"를 설명 가능한 형태로 정리.  
    - Dockerfile은 직접 작성해야 한다.
