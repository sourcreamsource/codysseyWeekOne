## 🟢 1. 백엔드와 nginx의 역할 차이  

#### ⚫️ **AWS EC2**  
프로그램들을 올려서 실행할 수 있는 **서버 컴퓨터 역할**을 한다.  

#### ⚫️ **백엔드(Node.js, Django 등)**  
로그인, 회원가입, 게시글 저장, 주문 처리처럼 **실제 서비스 기능과 비즈니스 로직**을 담당한다.  

#### ⚫️ **nginx**  
들어온 웹 요청을 먼저 받아서 정적 파일을 직접 응답하거나, 뒤쪽 서버(백엔드 서버)로 전달하는 일명 **✅ 앞단 웹서버 프로그램**이다.  



<br><br>

## 🟢 2. 구체적으로 nginx가 하는 4가지  

### 🟡 1. 정적 파일 직접 응답  

- `index.html`, `main.css`, `logo.png` 같은 파일  
- 백엔드(Django, Node.js 등)는 DB 조회, 로직 처리가 목적  
- 이런 파일까지 백엔드가 응답하면 **완전한 낭비**  
- nginx가 직접 디스크에서 파일 꺼내서 줌 → 백엔드 코드 한 줄도 안 탐  

<br>

### 🟡 2. 로드밸런싱 (트래픽 분산)  

- 유저 10만 명이 동시 접속하면 백엔드 1개로 못 버팀  
- 백엔드를 10개 켜두고, nginx가 트래픽을 골고루 나눠줌  
- 백엔드가 이걸 직접 하려면? **불가능**. 백엔드는 자기 요청만 처리함  

<br>

### 🟡 3. SSL 처리 (HTTPS)  

- HTTPS 암호화/복호화는 CPU 비용이 큼  
- nginx가 앞단에서 SSL 다 처리하고, 백엔드한테는 **평문 HTTP로 전달**  
- 백엔드는 SSL 신경 안 써도 됨  

<br>

### 🟡 4. 보안 / 요청 필터링  

- 백엔드 서버 IP/포트를 **외부에 직접 노출 안 해도 됨**  
- DDoS, 악성 요청, Rate Limiting 등을 앞단에서 처리  
- 백엔드가 이걸 직접 하면 코드가 비즈니스 로직이 아니라 인프라 코드가 됨  



<br><br>

## 🟢 결론  

> 백엔드 없애고 nginx만으론 앱 못 만듦. nginx 없애고 백엔드만으론 **실서비스 운영 못 함.**  
> 둘은 대체 관계가 아니라 **역할 분리** 관계임.
